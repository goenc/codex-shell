目的:
既存ソフトへ UI編集拡張を最小差分で移植する。
既存挙動を壊さず、必須10項目を漏れなく移植することを最優先とする。

実装方針:
- src/tools/ui_edit/ を機能単位で追加する
- 既存側の変更は入口接続のみ（原則1〜3ファイル以内）
- 既存コードからは tools::ui_edit::api のみ参照し、内部実装へ直接依存しない
- 既存のコマンドIDとUI定義形式の互換性を維持する
- 無関係なリファクタリングは禁止

必須移植項目（全10項目を必須）:
1. メイン画面オブジェクトのクリック選択
2. 選択オブジェクトの赤枠ハイライト表示
3. 修飾キー付きの複数選択（Shift/Ctrl/Cmd）
4. 選択オブジェクトのドラッグ移動（複数同時移動含む）
5. UI編集ON中のメイン操作ロック（main コマンド無効化ガード）
6. UI編集ON状態のロック表示文言（UI編集: ON (主画面操作ロック)）
7. runtime/ui/live/ui.json 未配置時の自動生成フォールバック
8. UiDefinition / UiObject などのデータモデルを main.rs 側に新規実装
9. UI定義の整合補完処理（ensure_integrity で不足オブジェクト補充）
10. command.rs への #![allow(dead_code)] 追加（警告抑制）

実装要件（10項目の取りこぼし防止）:
- 項目1〜4:
  - クリック選択、修飾キー複数選択、選択ハイライト、ドラッグ移動を同一画面で実行できること
  - 複数選択時は主選択と副選択を区別して描画できること
- 項目5〜6:
  - UI編集ON時は main 操作コマンドを dispatch しないこと
  - ロック表示文言は UI上で明示し、ON時のみ表示すること
- 項目7:
  - runtime/ui/live/ui.json が無い場合はエラー終了せず初期UI定義を生成して保存すること
  - 生成後は通常読込フローへ合流すること
- 項目8〜9:
  - UiDefinition / UiObject などのモデルを移植先 main.rs 側へ定義すること
  - ensure_integrity を持たせ、不足スクリーン/不足オブジェクトを補完すること
- 項目10:
  - src/tools/ui_edit/command.rs 先頭へ #![allow(dead_code)] を追加すること

変更許可:
- src/tools/**
- 入口接続に必要な既存最小ファイルのみ

変更禁止:
- 実装範囲外の既存ロジック変更
- 既存コマンド仕様の破壊
- 依存クレートの不要追加

依存クレート要件（移植先プロジェクト）:
- 実装開始前に移植先 Cargo.toml を確認する
- 以下を必要クレートとし、未導入が1つでもあれば実装を開始しない
  - eframe（egui を含む。UI編集画面の描画とViewport制御に必須）
  - serde（derive を有効化。UI定義のシリアライズ/デシリアライズに必須）
  - serde_json（runtime/ui/live/ui.json の読込/保存に必須）
- 推奨（既存実装が Result + Context 方式の場合のみ）
  - anyhow
- 上記以外のGUIクレート追加（例: iced / gtk / slint）は禁止
- バージョンは移植先プロジェクトの既存バージョンを優先し、互換を崩す更新は行わない

実装中止条件:
- 必要クレート（eframe / serde(derive) / serde_json）が未導入
- 既存依存との競合で同時利用が困難
- バージョン制約により互換性を満たせない
- 上記に該当した場合は実装を開始せず、不足クレート名と中止理由を記録して終了する

受け入れ条件:
1. debug build 成功
2. 拡張OFF時の既存挙動が維持される
3. 拡張ON時にUI編集画面が表示できる
4. 編集→保存→再起動で反映が維持される
5. 必須移植項目1〜10がすべて満たされる
6. 未知UIコマンドが発生しない

最終確認チェック（完了前に必須）:
- 10項目の実装有無を項目番号付きで自己点検し、全項目 OK を記録する
- runtime/ui/live/ui.json を一度削除した状態で起動し、自動生成フォールバックを確認する
- UI編集ON時に主画面操作がロックされ、指定文言が表示されることを確認する
- クリック選択、複数選択、ドラッグ移動、ハイライト描画を実機確認する

失敗時:
- 入口接続差分をロールバックして既存経路へ戻す
- 失敗内容は再現手順付きで記録する
