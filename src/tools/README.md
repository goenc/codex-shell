# tools 構成ルール

このディレクトリは機能単位の拡張を配置する。

各機能は `src/tools/<機能名>/` に分離し、公開入口を `api.rs` に集約する。
既存コードからは `api.rs` のみ参照し、内部実装ファイルへの直接参照は行わない。

`ui_edit` 機能は次の構成で管理する。

- `api.rs`: 既存側が利用する公開インターフェース
- `command.rs`: UIコマンド定義
- `editor.rs`: UI編集画面の描画ロジック

既存挙動を維持するため、機能追加時は次を守る。

- 既存変更は `main.rs` のモジュール参照差し替えを最小化する
- 既存のコマンドIDやUI定義形式は互換維持する
- 機能が無効でも既存動作が変わらないことを確認する
